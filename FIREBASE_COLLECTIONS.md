# Firebase Collections for ESI/PF File Generator

## Collections to Add in Firebase Firestore

### 1. `companies`

Stores company information including EPF and ESI percentages.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | string | Firebase Auth user ID (owner of this company) |
| `name` | string | Company name |
| `epfEmployeePercent` | number | Employee EPF % (e.g., 12) |
| `esiEmployeePercent` | number | Employee ESI % (e.g., 0.75) |
| `epfEmployer8_33Percent` | number | Employer EPF 8.33% component |
| `epfEmployer3_67Percent` | number | Employer EPF 3.67% component |
| `esiEmployerPercent` | number | Employer ESI % (e.g., 3.25) |

**Document ID:** Auto-generated by Firestore

---

### 2. `user_percentages`

Stores custom percentage options per user. Each user has their own list.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | string | Firebase Auth user ID |
| `value` | number | Percentage value (e.g., 12, 8.33) |

**Document ID:** `{userId}_{value}` (e.g., `abc123_12.00`) to prevent duplicates.

**Required Index:** Composite index on `userId` (Ascending) + `value` (Ascending).

---

### 3. `employees`

Stores employee details. Each employee belongs to a company.

| Field | Type | Description |
|-------|------|-------------|
| `companyId` | string | Reference to companies document ID |
| `name` | string | Employee full name |
| `uan` | string | Universal Account Number (EPF) |
| `ipNumber` | string | Insurance Permanent Number (ESI) |
| `salaryPerDay` | number | Daily wage |

**Document ID:** Auto-generated by Firestore

**Required Index:** Composite index on `companyId` (Ascending) + `name` (Ascending) for the employees query.

---

### 4. `transactions`

Stores monthly transaction headers. Each document represents one month's payroll for a company.

| Field | Type | Description |
|-------|------|-------------|
| `companyId` | string | Reference to companies document ID |
| `month` | number | Month (1-12) |
| `year` | number | Year (e.g., 2026) |

**Document ID:** Auto-generated by Firestore

**Subcollection:** `entries` (see below)

**Required Index:** Composite index on `companyId` (Ascending) + `year` (Descending) + `month` (Descending).

---

### 5. `transactions/{transactionId}/entries` (Subcollection)

Stores individual employee entries for each month's transaction.

| Field | Type | Description |
|-------|------|-------------|
| `transactionId` | string | Parent transaction document ID |
| `employeeId` | string | Reference to employees document ID |
| `employeeName` | string | Employee name (denormalized for display) |
| `uan` | string | Employee UAN |
| `ipNumber` | string | Employee IP Number |
| `salaryPerDay` | number | Daily wage |
| `noOfDays` | number | Days worked in the month |
| `totalSalary` | number | salaryPerDay × noOfDays |
| `employeeEpf` | number | Employee EPF 12% amount |
| `employeeEsi` | number | Employee ESI 0.75% amount |
| `employerEpf8_33` | number | Employer EPF 8.33% amount |
| `employerEpf3_67` | number | Employer EPF 3.67% amount |
| `employerEsi` | number | Employer ESI 3.25% amount |
| `grossSalary` | number | totalSalary - employeeEpf - employeeEsi |

**Document ID:** Auto-generated by Firestore

**Required Index:** Composite index on `transactionId` (Ascending) + `employeeName` (Ascending) for ordering entries.

---

## Firebase Console Setup

1. **Create Firestore Database** (if not exists)
   - Go to Firebase Console → Firestore Database → Create database
   - Start in test mode for development (add security rules for production)

2. **Create Composite Indexes**
   - Go to Firestore → Indexes → Composite
   - Add:
     - Collection: `companies` | Fields: `userId` Asc, `name` Asc
     - Collection: `user_percentages` | Fields: `userId` Asc, `value` Asc
     - Collection: `employees` | Fields: `companyId` Asc, `name` Asc
     - Collection: `transactions` | Fields: `companyId` Asc, `year` Desc, `month` Desc

3. **Add Firebase to Flutter**
   - Run `flutterfire configure` to generate `firebase_options.dart`
   - Or manually add Firebase project in `google-services.json` (Android) and `GoogleService-Info.plist` (iOS)
